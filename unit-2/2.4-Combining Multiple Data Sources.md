# 2.4 Combining Multiple Data Sources

## Learning Outcomes

After completing this topic, students will be able to:

* Understand the need for combining multiple data sources.
* Differentiate between **Append** and **Merge** operations in Power Query.
* Create and manage relationships between tables.
* Apply data integration techniques in Power BI.
* Consolidate data from different business systems for reporting and analysis.

---

# Introduction

In real-world organizations, data is rarely stored in a single location. Sales data may be stored in SQL Server, customer information in Excel, product details in SAP, and employee records in SharePoint.

To generate meaningful reports, Power BI combines data from these different sources into a single data model.

### Definition

**Data Consolidation** is the process of combining data from multiple sources into a unified dataset for analysis and reporting.

---

# Why Combine Multiple Data Sources?

Organizations use different applications for different business functions.

| Department | Data Source      |
| ---------- | ---------------- |
| Sales      | SQL Server       |
| Finance    | Oracle Database  |
| HR         | Excel            |
| Inventory  | SAP ERP          |
| Marketing  | Google Analytics |

Power BI integrates all these sources into a single dashboard.

---

# Benefits of Combining Data

* Creates a unified view of the business.
* Eliminates data silos.
* Improves reporting accuracy.
* Supports better decision-making.
* Enables cross-functional analysis.

# Combining Data in Power BI

Power BI mainly provides two methods:

1. **Append Queries**
2. **Merge Queries**

---

# 1. Append Queries

## Definition

**Append** combines two or more tables by adding rows one below another.

It is similar to the SQL **UNION** operation.

Use Append when the tables have the **same columns**.

---

## Example

### Sales – January

| Product | Sales |
| ------- | ----: |
| Laptop  |    50 |
| Mobile  |    80 |

### Sales – February

| Product | Sales |
| ------- | ----: |
| Laptop  |    60 |
| Mobile  |    90 |

### After Append

| Product | Sales |
| ------- | ----: |
| Laptop  |    50 |
| Mobile  |    80 |
| Laptop  |    60 |
| Mobile  |    90 |

The rows from February are added below January.

---

## Steps to Append Tables

1. Open **Power Query Editor**.
2. Select **Home → Append Queries**.
3. Choose:

   * Two Tables, or
   * Three or More Tables.
4. Select the tables.
5. Click **OK**.

---

## When to Use Append

Use Append when:

* Monthly sales files need to be combined.
* Regional datasets have the same structure.
* Annual reports are created from monthly data.

---

### Business Example

A company stores sales separately:

* January.xlsx
* February.xlsx
* March.xlsx

Append combines them into a single **Annual Sales** table.

---

# Advantages of Append

* Combines multiple tables quickly.
* Suitable for historical data.
* Easy to maintain.
* Supports data consolidation.

---

# 2. Merge Queries

## Definition

**Merge** combines tables by matching common columns (keys).

It is similar to a SQL **JOIN**.

---

## Example

### Customers Table

| Customer ID | Customer Name |
| ----------- | ------------- |
| 101         | Ravi          |
| 102         | Anita         |
| 103         | Rahul         |

### Orders Table

| Order ID | Customer ID | Amount |
| -------- | ----------- | -----: |
| 5001     | 101         |  12000 |
| 5002     | 102         |  18000 |
| 5003     | 103         |  22000 |

### After Merge

| Customer Name | Order ID | Amount |
| ------------- | -------- | -----: |
| Ravi          | 5001     |  12000 |
| Anita         | 5002     |  18000 |
| Rahul         | 5003     |  22000 |

---

## Steps to Merge Tables

1. Open **Power Query Editor**.
2. Select **Home → Merge Queries**.
3. Choose the two tables.
4. Select the common column (e.g., Customer ID).
5. Select the Join Type.
6. Expand the merged columns.

---

# Join Types in Merge

| Join Type        | Description                                                                    |
| ---------------- | ------------------------------------------------------------------------------ |
| Left Outer Join  | Returns all rows from the first table and matching rows from the second table. |
| Right Outer Join | Returns all rows from the second table and matching rows from the first table. |
| Full Outer Join  | Returns all rows from both tables, matching where possible.                    |
| Inner Join       | Returns only matching rows from both tables.                                   |
| Left Anti Join   | Returns rows in the first table with no match in the second.                   |
| Right Anti Join  | Returns rows in the second table with no match in the first.                   |

---

## Business Example

A company wants customer names in the sales report.

* Customer details → Excel
* Orders → SQL Server

Merge combines the two datasets using **Customer ID**.

---

# Append vs Merge

| Feature         | Append              | Merge                          |
| --------------- | ------------------- | ------------------------------ |
| Purpose         | Add rows            | Combine columns                |
| SQL Equivalent  | UNION               | JOIN                           |
| Table Structure | Same columns        | Related tables with common key |
| Result          | More rows           | More columns                   |
| Example         | Monthly sales files | Customers + Orders             |

# 3. Managing Relationships

## What is a Relationship?

A **relationship** connects two or more tables using a common field (key).

Relationships allow Power BI to analyze related data without duplicating it.

---

## Why Relationships are Needed

Example:

**Customers**

| Customer ID | Name |
| ----------- | ---- |
| 101         | Ravi |

**Orders**

| Order ID | Customer ID | Amount |
| -------- | ----------- | -----: |
| 5001     | 101         |  12000 |

Using **Customer ID**, Power BI links the two tables.

---

# Types of Relationships

## 1. One-to-One (1:1)

Each record in one table matches only one record in another table.

Example:

| Employee     | Employee Profile |
| ------------ | ---------------- |
| One employee | One profile      |

---

## 2. One-to-Many (1:*)

One record in the first table relates to many records in the second table.

Example:

One customer can place many orders.

```text
Customer
   │
   ├── Order 1
   ├── Order 2
   └── Order 3
```

This is the most common relationship in Power BI.

---

## 3. Many-to-Many (*:*)

Multiple records in one table relate to multiple records in another table.

Example:

Students and Courses.

One student studies many courses, and one course has many students.

---

# Creating Relationships

Steps

1. Open **Model View**.
2. Drag the common column from one table to the matching column in another table.
3. Power BI creates the relationship.
4. Verify:

   * Cardinality
   * Cross-filter direction
   * Active relationship

---

# Best Practices for Relationships

* Use unique primary keys.
* Avoid duplicate keys.
* Prefer a **Star Schema**.
* Avoid unnecessary many-to-many relationships.
* Ensure data types match before creating relationships.

## Classroom Practical (2 Hours)

Students will perform the following activities:

1. Import two monthly sales Excel files and **Append** them into a single table.
2. Import a **Customers** table and **Orders** table and **Merge** them using **Customer ID**.
3. Create relationships between **Products**, **Customers**, and **Sales** tables in **Model View**.
4. Verify relationship cardinality (One-to-Many).
5. Build a simple report displaying:

   * Total Sales by Customer
   * Product-wise Sales
   * Regional Sales
6. Observe how relationships enable filtering and cross-analysis across multiple tables.

This exercise helps students understand how organizations consolidate data from different sources into a unified Power BI data model for reporting and analytics.
