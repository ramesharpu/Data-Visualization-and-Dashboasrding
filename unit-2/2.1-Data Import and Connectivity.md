# Introduction

Before creating reports and dashboards, Power BI must first **connect to the required data**. Organizations store their data in various formats such as Excel files, databases, cloud applications, and websites.

Power BI provides more than **100 built-in connectors** to connect to these different data sources.

The process of bringing data into Power BI is called **Data Import and Connectivity**.

---

# What is Data Import?

**Data Import** is the process of connecting Power BI to one or more data sources and loading the required data into Power BI Desktop for analysis and reporting.

### Definition

> **Data Import** is the process of retrieving data from different sources and loading it into Power BI for transformation, modeling, visualization, and reporting.

---

# Why is Data Import Important?

Organizations use different software to store data.

For example:

| Department | Data Source      |
| ---------- | ---------------- |
| Sales      | SQL Server       |
| HR         | Excel            |
| Finance    | Oracle Database  |
| Marketing  | Google Analytics |
| Inventory  | SAP ERP          |

Power BI combines all these data sources into one integrated report.

---

# Types of Data Sources

Power BI supports many types of data sources.

## 1. Excel Files

Excel is one of the most commonly used data sources.

Supported formats:

* .xlsx
* .xls

### Example

A company stores monthly sales data in **Sales.xlsx**.

| Product | Sales     |
| ------- | --------- |
| Laptop  | ₹2,50,000 |
| Mobile  | ₹1,80,000 |
| Printer | ₹90,000   |

Power BI imports this Excel file and creates charts and dashboards.

---

## Steps to Connect Excel

1. Open Power BI Desktop.
2. Select **Home → Get Data → Excel**.
3. Browse and select the Excel file.
4. Choose the worksheet(s).
5. Click **Load** or **Transform Data**.

---

### Advantages

* Easy to connect
* Widely used
* Suitable for small and medium datasets

---

### Limitations

* Manual updates
* Large files reduce performance

---

# 2. CSV Files

CSV (Comma-Separated Values) is a plain text file containing tabular data.

Example:

```csv

Ravi,Sales,45000
Anita,HR,50000
Rahul,Finance,60000
```

---

## Steps to Connect CSV

1. Home
2. Get Data
3. Text/CSV
4. Select CSV file
5. Preview data
6. Load or Transform

---

### Advantages

* Lightweight
* Easy to exchange data
* Supported by almost every application

---

### Limitations

* No formatting
* No formulas
* Data types may need adjustment

# Import Mode vs DirectQuery Mode

When connecting to databases, Power BI offers two connection modes.

## 1. Import Mode

### Definition

Power BI copies data into its own internal storage.

After importing, reports use the copied data.

---

### Workflow

```text
Database
     │
Import
     │
Power BI Memory
     │
Reports
```

---

### Advantages

* Very fast
* Supports all Power BI features
* Works offline
* Better report performance

---

### Disadvantages

* Data becomes outdated until refreshed

---

### Example

Sales data is imported every morning.

The dashboard shows today's morning data until the next refresh.

---

## 2. DirectQuery Mode

### Definition

Power BI does not store the data.

Every chart sends a live query to the database.

---

### Workflow

```text
Dashboard
      │
Live Query
      │
Database
      │
Results Returned
```

---

### Advantages

* Real-time data
* Suitable for huge databases
* No duplicate storage

---

### Disadvantages

* Slower than Import Mode
* Depends on database performance
* Some Power BI features are limited

---

### Example

A bank displays live ATM transactions.

Every chart shows the latest transaction data directly from the SQL Server.

---

# Comparison of Import and DirectQuery

| Feature                 | Import                  | DirectQuery                |
| ----------------------- | ----------------------- | -------------------------- |
| Data Stored in Power BI | Yes                     | No                         |
| Speed                   | Very Fast               | Depends on database        |
| Offline Access          | Yes                     | No                         |
| Real-Time Data          | No                      | Yes                        |
| Best For                | Small & Medium datasets | Large enterprise databases |
| Refresh Required        | Yes                     | No (queries live data)     |

---

# Which Mode Should You Choose?

### Use Import Mode when:

* Dataset is small to medium.
* High performance is required.
* Data does not change frequently.
* Offline analysis is needed.

---

### Use DirectQuery when:

* Data changes every minute.
* Dataset is very large.
* Real-time dashboards are required.
* Data must remain in the source database.

---

# Data Refresh Concepts

## What is Data Refresh?

**Data Refresh** updates Power BI reports with the latest data from the original source.

Without refresh, reports may show outdated information.

---

## Types of Data Refresh

### 1. Manual Refresh

The user refreshes the data manually.

**Steps**

Home → Refresh

**Example**

An analyst refreshes monthly sales data after receiving the latest Excel file.

---

### 2. Scheduled Refresh

Power BI Service automatically refreshes data at specified times.

Examples:

* Every day at 8:00 AM
* Every hour
* Every Monday

**Example**

A sales dashboard refreshes automatically every morning before office hours.

---

### 3. Automatic Refresh (DirectQuery)

Since DirectQuery uses live queries, visuals retrieve the latest data whenever they are opened or interacted with.

**Example**

A hospital dashboard displays the latest patient admissions in near real time.

---

# Refresh Workflow

```text
Data Source
      │
      ▼
Power BI Refresh
      │
      ▼
Updated Dataset
      │
      ▼
Reports
      │
      ▼
Dashboards
```
